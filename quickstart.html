SDK quickstart

A docs-aligned activation page for getting the first project instrumented and sending the first event into TokenLens.

Project activation

Install the SDK, capture one request, then verify it in the overview.

The quickstart focuses on the minimum developer path: add an API key, instrument request metadata, send a test prompt, and confirm that aggregated metrics appear in the project dashboard.

Current context
  • Time to first event target: under 10 minutes
  • Scope fields: provider, model, route, status, tokens, latency, tags
  • Verification path: projects → overview → recent requests
  • Use this page as a static placeholder for the eventual docs-backed app route

Quickstart steps

Insight first, details second

1. Create a project-scoped key

Generate a server-side ingestion key and copy it into the app runtime or local development environment.

2. Wrap one production-like route

Start with the route that carries the most traffic or cost so the overview becomes useful quickly.

3. Capture normalized metadata

Send request ID, route, provider, model, latency, token counts, estimated cost, tags, and final status.

4. Verify the first event

Use recent requests and the aggregate cards in the overview to confirm ingestion and expected dimensions.

Implementation checklist

Suggested MVP instrumentation
  • Attach a stable request ID for drill-downs.
  • Record provider and model on every call.
  • Send input and output token counts separately.
  • Track latency in milliseconds for percentile views.
  • Tag environment, route, and workload type.

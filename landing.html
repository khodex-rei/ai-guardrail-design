SDK integration 
from openai import OpenAI
from tokenlens import TokenLens

client = TokenLens.wrap_openai(
  OpenAI(api_key=os.environ["OPENAI_API_KEY"]),
  api_key=os.environ["TOKENLENS_API_KEY"],
  project="support-copilot",
  environment="production"
)

response = client.responses.create(
  model="gpt-4.1-mini",
  input="Summarize this customer thread"
)

FAQ

Common questions from teams evaluating the SDK.

What does TokenLens capture by default?

The default metrics path focuses on request counts, token usage, estimated cost, latency, provider, model, status, and project metadata so teams can monitor usage without storing unnecessary raw payloads.

How fast can we get value?

The primary onboarding goal is first event received. Create a project, generate an API key, wrap a client, send a request, and confirm the event in the dashboard.

Which providers fit the initial workflow?

The marketing and docs direction centers on OpenAI-compatible and Anthropic-compatible integrations first, with model and provider analytics surfaced in one dashboard.

How does pricing scale?

Plans primarily scale by tracked events, retention depth, team or project scope, and access to alerts, exports, and more advanced analytics workflows.
